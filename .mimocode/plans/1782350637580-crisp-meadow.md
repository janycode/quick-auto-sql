# 项目综合分析报告

## 一、安全问题（CRITICAL）

### 1. 硬编码管理员密码
- **文件**: `server/src/services/auth.ts:15`
- **问题**: `DEFAULT_PASSWORD = 'jy19900903'` 硬编码在源码中
- **建议**: 首次运行时生成随机密码，或要求用户在初始化时设置

### 2. SMTP 凭据提交到 Git
- **文件**: `server/.env`
- **问题**: `SMTP_PASS=guluilbxlczocbcb` 等真实凭据存在于仓库中（即使 .gitignore 可能已排除，历史记录中仍有）
- **建议**: 立即轮换 SMTP 授权码，确保 .env 完全不在 Git 历史中

### 3. 默认加密密钥已公开
- **文件**: `server/.env`, `server/src/config/index.ts`
- **问题**: `ENCRYPT_KEY=quick-auto-sql-default-key` 是默认值且已公开，所有用此密钥加密的数据库密码可被解密
- **建议**: 每次部署生成唯一密钥，移除默认值

### 4. 弱密码哈希 — 无盐 SHA256
- **文件**: `server/src/services/auth.ts:57-59`
- **问题**: `SHA256(password)` 无盐，易受彩虹表和暴力破解
- **建议**: 迁移到 bcrypt 或 argon2

### 5. SQL 注入风险 — EXPLAIN 拼接
- **文件**: `server/src/services/query.ts:246`
- **问题**: `EXPLAIN ${sql}` 直接拼接 SQL 字符串
- **建议**: 严格校验只允许 SELECT/INSERT/UPDATE/DELETE 通过 explainQuery

### 6. SQL 禁用关键词绕过
- **文件**: `server/src/routes/query.ts:20-27`
- **问题**: `trimmedSql.startsWith(keyword)` 只检查首词，`; DROP TABLE` 可绕过
- **建议**: 按 `;` 分割后逐条检查首关键词

## 二、高优先级问题（HIGH）

### 7. JSON 文件读写无锁 — 并发竞争
- **文件**: 所有 JsonStore 调用处
- **问题**: readFileSync + writeFileSync 无原子保证，并发请求可能丢数据
- **建议**: 引入互斥锁/信号量，或迁移到 SQLite

### 8. 连接池未在进程退出时清理
- **文件**: `server/src/services/connection.ts:9,23-43`
- **问题**: `poolCache` 无 SIGTERM/SIGINT 清理逻辑
- **建议**: 添加优雅关闭处理

### 9. 无优雅关闭机制
- **文件**: `server/src/app.ts`
- **问题**: 无 `process.on('SIGTERM')` 处理，Docker 停止时会中断进行中的请求
- **建议**: 添加优雅关闭：关闭 Express → 排空连接池 → 退出

### 10. CORS 完全开放
- **文件**: `server/src/app.ts:20`
- **问题**: `cors()` 无限制，允许任何来源访问 API
- **建议**: 生产环境配置白名单

### 11. CSRF 保护在开发环境被禁用
- **文件**: `server/src/middleware/security.ts:50-55`
- **问题**: 非生产环境完全跳过 CSRF 校验，且 CSRF 中间件未在 app.ts 中挂载
- **建议**: 至少在生产环境挂载并启用

### 12. AI API Key 明文存储
- **文件**: `server/src/store/json-store.ts:498-586`
- **问题**: AI 配置中的 API Key 未加密（MySQL 密码有 AES 加密，AI Key 没有）
- **建议**: 使用相同 ENCRYPT_KEY 机制加密 AI API Key

### 13. 速率限制可伪造
- **文件**: `server/src/middleware/security.ts:68-120`
- **问题**: 基于 `req.ip`（可被 X-Forwarded-For 伪造），重启后状态丢失
- **建议**: 使用 Redis 存储或固定信任最左 X-Forwarded-For

### 14. MySQL 错误码/处理函数重复定义 5 次
- **文件**: `utils/mysql-error.ts`, `services/query.ts`, `services/connection.ts`, `services/database.ts`, `middleware/error-handler.ts`
- **问题**: 已有共享文件但未被引用，各处复制粘贴
- **建议**: 统一从 `utils/mysql-error.ts` 导入
