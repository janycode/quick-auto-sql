# 项目深度分析报告（第三轮）

## 一、后端问题

### A. 安全与健壮性

| 严重度 | 文件:行 | 问题 | 建议 |
|--------|---------|------|------|
| MEDIUM | `services/feedback.ts:34-37` | `writeFeedbacks` 使用 `fs.writeFileSync` 非原子写入 | 改用 tmp+rename 模式 |
| MEDIUM | `services/pay.ts:58-63` | `writeOrders` 使用 `fs.writeFileSync` 非原子写入 | 同上 |
| MEDIUM | `services/auth.ts:82-85` | `writeJson` 使用 `fs.writeFileSync` 非原子写入 | 同上 |
| MEDIUM | `services/ai.ts:310-378` | `generateSqlStream` 中 `fetch` 无超时设置，AI API 无响应时会永久挂起 | 添加 AbortController + 超时（如 60s） |
| MEDIUM | `services/ai.ts:130-148` | `testAiConnection` 同样无超时 | 同上 |
| LOW | `middleware/auth.ts:26-29` | Token 从 query/body 传递存在泄露风险（URL 日志、Referer 头） | 移除 query/body token 支持，仅保留 Header |
| LOW | `services/ai.ts:413-414` | SSE 流解析中 `catch {}` 吞掉所有异常，包括网络中断 | 至少 log 警告 |
| LOW | `routes/ai.ts:194` | `/generate` 路由未挂载 `requireAuth`（依赖 optionalAuth），未登录用户也能调用 | 添加 requireAuth |

### B. 代码质量

| 严重度 | 文件:行 | 问题 | 建议 |
|--------|---------|------|------|
| MEDIUM | `services/quota.ts:118` | `historyUsed: 0` 硬编码，从未计算真实值 | 从 queryHistoryStore 按用户统计 |
| MEDIUM | `store/json-store.ts:649-653` | `readActiveAiConfig()` 永远返回 null，死代码 | 删除 |
| LOW | `store/json-store.ts:252-255` | `withLock` 同步版本是空操作，锁未生效 | 移除或改为真正串行化 |
| LOW | `services/feedback.ts` / `services/pay.ts` | 未使用 JsonStore，自己实现 read/write 逻辑 | 统一使用 JsonStore |
| LOW | `config/index.ts:5` | `raw === null` 永远不会触发 | 移除 null 检查 |
| LOW | `services/quota.ts:62` | `Partial<IUsageRecord>` 类型已修复但仍有多处 `as any` | 完善类型定义 |

### C. 架构设计

| 问题 | 说明 |
|------|------|
| **JSON 文件无备份** | 12 个 JSON 文件无自动备份机制，磁盘损坏即丢失全部数据 |
| **会话存储性能** | `findSession` 每次调用 `pruneExpiredSessions()` 读写全量文件，高并发下成为瓶颈 |
| **AI 配置读写放大** | `getAiConfigList` 每次调用读取 ai-config.json + users.json，多用户并发时 I/O 压力大 |
| **无请求 ID** | 日志中无法关联同一请求的多条 log，排查问题困难 |
| **无 API 版本化** | 所有接口都在 `/api/` 下，未来 breaking change 无法平滑迁移 |

## 二、前端问题

### A. TypeScript 错误（pre-existing，6 处）

| 文件 | 行 | 错误 |
|------|-----|------|
| `FeedbackAdmin.vue` | 45,50,110,115 | `row.type`/`row.status` 隐式 any |
| `FeedbackAdmin.vue` | 76,79 | `DefaultRow` 不可赋值给 `IFeedback` |
| `FeedbackAdmin.vue` | 155 | string 不可赋值给 Element Plus tag type |
| `FeedbackAdmin.vue` | 301,326 | `res.code` 不存在于 AxiosResponse |
| `Home.vue` | 560 | `res.code` 不存在于 AxiosResponse |
| `SqlEditor.vue` | 2014 | `string | undefined` 不可赋值给 `string` |

### B. 性能与 UX

| 问题 | 文件 | 说明 |
|------|------|------|
| **localStorage 溢出** | `workspace.ts:72-75` | `queryResult` 深监听序列化到 localStorage，大结果集（1000 行）可能超 5MB 限制 |
| **keep-alive 内存泄漏** | `App.vue:2-6` | 所有页面组件被 keep-alive 缓存，Monaco Editor 实例不会销毁，内存持续增长 |
| **深拷贝 watch** | `workspace.ts:79-86` | 多个 `{ deep: true }` watch 每次变更都序列化整个对象，频繁操作时性能差 |
| **SSE 无重连机制** | `sse.ts` | SSE 连接断开后不自动重连，用户需手动重试 |
| **对话历史无限增长** | `workspace.ts:66-68` | `conversationMessages` 无上限，localStorage 可能被撑爆 |
| **无 loading 状态管理** | 多处 | AI 生成/优化的 loading 状态分散在 workspace store 中，无统一管理 |

### C. 代码质量

| 问题 | 文件 | 说明 |
|------|------|------|
| **request.ts 过于复杂** | `request.ts:307行` | 响应拦截器包含 MySQL 友好错误、未登录处理、429 配额、蒙版弹窗等大量逻辑，难以维护 |
| **窗口全局变量** | `request.ts:177-178` | `(window as any)[flagKey]` 用于防重复弹窗，不优雅 |
| **类型不一致** | `FeedbackAdmin.vue` | Element Plus Table 的 `row` 类型未正确标注 |
| **API 返回类型缺失** | `request.ts:230` | 拦截器 `return data` 丢失了 AxiosResponse 类型信息 |
